# 目录拣选模式

## 1. 背景

LocalManager 的根目录已通过 SAF `OpenDocumentTree` 一次性授权。许多功能只需要在
**已授权目录树内** 选文件或目录，而不应：

- 反复弹出系统文件选择器；
- 在选文件过程中误打开播放器、阅读器等，打断任务；
- 与日常「待处理列表」（拷贝、压缩等）互相污染。

因此将「临时切到目录 Tab、只能拣选、完成后回到原界面」抽象为应用级标准能力：**目录拣选模式**（Directory Pick Mode）。

## 2. 目标

| 目标 | 说明 |
|------|------|
| 复用 | 留言板附件、Git 分享、批量上传等共用同一套状态机与 UI 约束 |
| 隔离 | 每次拣选使用独立 `pickList`，不占用日常 `pendingList` |
| 可恢复 | 发起方 Tab/会话在拣选期间保持；取消或完成后回到发起方 |
| 受限 | 拣选模式下禁止打开/播放/编辑，仅允许浏览与勾选 |
| 可扩展 | 通过 `PickPolicy` 配置单选/多选、是否允许目录等 |

## 3. 核心概念

### 3.1 DirectoryPickRequest（拣选请求）

由任意功能发起，描述「要什么」和「完了怎么办」。

建议类型（Kotlin `sealed class`）：

```
DirectoryPickRequest
├── id: String                    // 会话 ID，用于回调关联
├── purpose: DirectoryPickPurpose // 用途，决定文案与策略
├── returnTab: MainTab            // 完成后返回的 Tab
├── policy: DirectoryPickPolicy   // 选择规则
├── onComplete: (DirectoryPickResult) -> Unit
└── onCancel: () -> Unit
```

`DirectoryPickPurpose` 示例：

| 值 | 场景 | 完成按钮文案 |
|----|------|--------------|
| `BULLETIN_ATTACHMENT` | 家庭网络留言板附件 | 发送到留言板 |
| `GIT_SHARE` | 分享到 Git（未来） | 添加到分享 |
| `GENERIC` | 其它功能 | 确定 |

### 3.2 DirectoryPickPolicy（拣选策略）

```
DirectoryPickPolicy
├── allowFiles: Boolean = true
├── allowDirectories: Boolean = true   // 是否允许把整目录作为一项
├── multiSelect: Boolean = true
├── maxItems: Int? = null              // null = 不限制
└── allowedExtensions: Set<String>?    // null = 不限制
```

留言板附件默认：`allowFiles=true, allowDirectories=true, multiSelect=true`。

### 3.3 DirectoryPickResult（拣选结果）

```
DirectoryPickResult
├── requestId: String
├── items: List<DocumentFileModel>   // 文件或目录项
└── purpose: DirectoryPickPurpose
```

每项保留 SAF `uri`、`name`、`isDirectory`、可选 `size`（目录可在上传前再扫描）。

### 3.4 活跃会话

`FileBrowserApp` 顶层维护：

```
var activeDirectoryPick: DirectoryPickRequest? = null
var pickList: SnapshotStateList<DocumentFileModel>  // 仅本次会话
```

- `activeDirectoryPick != null` 时，目录 Tab 处于 **拣选模式**。
- `pickList` 与日常 `pendingList` **完全独立**；UI 可复用 `PendingListScreen` 的列表展示，但操作按钮不同。

## 4. 生命周期

```
[发起方] 用户触发「需要选文件」
    → 创建 DirectoryPickRequest + 清空 pickList
    → switchMainTab(DIRECTORY)
    → 目录进入拣选模式（顶栏/菜单/Tab 锁定）

[拣选中] 用户浏览目录、点选文件/目录加入 pickList
    → 单击文件：加入/移出 pickList（不打开）
    → 单击目录：进入子目录；可选「整目录加入」
    → 禁止：打开、播放、加密、删除、新建、换根目录等

[完成] 用户点「完成」
    → onComplete(DirectoryPickResult)
    → activeDirectoryPick = null；pickList 清空
    → switchMainTab(returnTab)

[取消] 用户点「取消」或返回键
    → onCancel()
    → 同上，但不调用 onComplete
```

### 4.1 返回键优先级（`BackHandler`）

拣选会话活跃时，返回键 **优先退出拣选**，而不是：

- 关闭留言板（`openBoardSession`）；
- 退出应用。

顺序建议：

1. 其它模态对话框
2. **`activeDirectoryPick != null` → 取消拣选并回到 `returnTab`**
3. 留言板内页 → 关闭留言板
4. 目录后退栈
5. 退出应用

### 4.2 底部 Tab 栏

拣选进行中：

- **锁定**除「目录」外的所有 Tab；点击提示「请先完成或取消文件选择」。
- 避免用户切到播放器后忘记拣选任务。

## 5. 目录 UI 约束（`DirectoryMode`）

`FileBrowserScreen` 增加参数：

```
directoryMode: DirectoryMode = NORMAL | PICK
pickPolicy: DirectoryPickPolicy?      // PICK 时必填
```

| 能力 | NORMAL | PICK |
|------|--------|------|
| 进入子目录 | ✓ | ✓ |
| 点文件打开/播放/阅读 | ✓ | ✗ → 切换 pickList |
| 上下文菜单 | 全量 | 白名单：加入/移出待选、整目录加入 |
| 待处理列表（拷贝/移动/压缩） | ✓ | ✗ 隐藏 |
| 新建/删除/重命名/GPG | ✓ | ✗ |
| 更换根目录 | ✓ | ✗ |
| 顶栏 | 默认 | 拣选条：用途文案 + 已选 N 项 + 待选列表 + 完成 + 取消 |

实现原则：**在 PICK 模式下不向 `onOpenFile` / `onRequestEpubView` 等传真实回调**，从结构上杜绝误触，而非仅弹 Toast 警告。

## 6. 待选列表 UI

复用现有待处理列表的列表 UI，通过参数区分模式：

```
PendingListScreen(
    items = pickList,
    mode = PendingListMode.DIRECTORY_PICK,
    title = purpose.title,
    confirmLabel = purpose.confirmLabel,
    onConfirm = { completePick() },
    onDismiss = { cancelPick() },
    // 不显示：压缩、加密、拷贝到当前目录 等
)
```

## 7. 标准发起 API（建议）

在 `FileBrowserApp` 内提供统一入口，避免各功能各自 `switchMainTab`：

```kotlin
fun startDirectoryPick(request: DirectoryPickRequest) {
    require(activeDirectoryPick == null) { "已有进行中的拣选会话" }
    pickList.clear()
    activeDirectoryPick = request
    switchMainTab(MainTab.DIRECTORY)
}

fun completeDirectoryPick() {
    val request = activeDirectoryPick ?: return
    request.onComplete(DirectoryPickResult(request.id, pickList.toList(), request.purpose))
    activeDirectoryPick = null
    pickList.clear()
    switchMainTab(request.returnTab)
}

fun cancelDirectoryPick() {
    activeDirectoryPick?.onCancel()
    activeDirectoryPick = null
    pickList.clear()
    activeDirectoryPick?.returnTab?.let { switchMainTab(it) }
}
```

各功能只构造 `DirectoryPickRequest`，不直接操作 Tab 与 `pickList`。

## 8. 首个消费者：留言板附件

见 `docs/家庭网络附件协议.md`（待写）中的 Android 拣选一节。流程：

1. `FamilyNetworkScreen` 点「添加附件」→ `startDirectoryPick(BULLETIN_ATTACHMENT, returnTab=FAMILY_NETWORK)`。
2. 用户拣选完成 → `onComplete` 将 `items` 交给 `FamilyNetworkManager` 上传队列。
3. 自动回到留言板；`openBoardSession` 未销毁。
4. 留言可附带文字后发送，或拣选完成即开始上传（实现时二选一，默认后者）。

下载仍使用固定目录 `{rootUri}/留言板附件/...`，与拣选模式无关。

## 9. 未来可复用场景

| 功能 | purpose | 策略差异 |
|------|---------|----------|
| 留言板附件 | `BULLETIN_ATTACHMENT` | 文件+目录，多选 |
| Git 分享 | `GIT_SHARE` | 可能仅文件 |
| 批量加入播放列表 | `PLAYLIST_ADD` | 仅音频扩展名 |
| 导入到某目录 | `IMPORT_TO` | 单文件 |

新功能只需新增 `DirectoryPickPurpose` 条目与文案，不必复制 Tab 切换逻辑。

## 10. 实现顺序建议

1. `DirectoryPickRequest` / `Policy` / `Result` 类型与 `FileBrowserApp` 状态机
2. `DirectoryMode.PICK` + 顶栏 + Tab 锁定 + `BackHandler`
3. `PendingListScreen` 拣选变体
4. 留言板附件作为第一个 `onComplete` 消费者
5. 附件上传协议与服务端（另文档）

## 11. 与现有组件关系

```
FileBrowserApp（持有 activeDirectoryPick、pickList）
    ├── MainTabsRootContent（Tab 锁定）
    ├── DirectoryTabRoute → FileBrowserScreen(directoryMode)
    ├── PendingListScreen（PICK 变体）
    └── FamilyNetworkScreen（发起 BULLETIN_ATTACHMENT）
```

日常 `pendingList` 行为不变；仅当 `activeDirectoryPick == null` 时目录为 `NORMAL` 模式。
