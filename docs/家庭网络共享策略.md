# 家庭网络共享策略

## 1. 介绍

本文构想一个家庭网络共享的方案。我们使用手机，常常需要一家人（也可以理解为一个
共同的组织）一起共享一组信息。网络运营商常常在公网上提供这种服务。但很多人并不
能接受信息直接被公开到网络上。

所以，我想着用家庭（或者组织）的局域网来完成这种信息共享，基本思路是：在家庭的
PC和手机上运行一个服务，注册mDNS服务端口，这样，当我们进入局域网的时候，就通过
这个服务发现互相的服务端口，我们可以连上这些用户，然后可以给它发信息，这些信息
可以是对话，文件，目录等内容。

这个思路要解决的第一个基本问题是：如何定义一个用户。

这说起来很简单，其实是最麻烦的，比如我们用一个id表示一个用户，那它就可以在设备
之间迁移，就可能出现两个设备上是同一个用户，又会出现不同用户使用相同的id（只要
没有一个中心就会这样），这些都很难做同步。

所以我们这样定义这个概念：任何一台设备都是一个留言板。

也就是说：你的id随便起名字，只代表你这个设备，你在网络里找到其他人了，你可以用
密码要求连上它，选择一个留言板，那你就可以在那个留言板上发消息。其他人和留言板
的宿主，也可以在留言板上发消息。这就在留言板上构成了一个群聊的局面。任何人只要
有密码（密码只是个泛泛的概念，可以是Token或者对称非对称密钥的形式），能连上这个
留言板，就在这个留言板上有身份，可以看到整个留言板的全部内容，可以在里面补充内
容。只有宿主才能维护留言板，其他人就只能“发言”，这种发言可以包括发文本信息，发
文件，这些信息可以被拷贝出来，也可以被管理员删除，备份等等。

连接密码我们可以提供两个，普通密码连上去就是普通用户，另一个密码连上来就是宿主，
这样我们可以作为远程宿主，管理服务器上的留言板。

这样，所有功能都可以自洽了：用户的概念只在一个留言板上生效，所有行为都是针对单
个设备的某个留言板的。

这种留言板服务，在PC上可以实现成一个服务器，在手机侧我们可以用ftp那个功能一样的
形式：开了界面才上线，退出就离线，这样仅做留言或者传输文件的时候才耗电，留言后
就可以离开了。本地留言板也可以设置不开，这样开了留言板服务仅仅就是连他方的留言
板就行了。

每个留言板还可以形成一个信息聚合体，未来我们可以让AI介入，帮我们管理这些聚合的
信息。

## 2. 设计思路

## 2.1 留言板的存储管理

留言板的内容保存在应用的私有存储中，每个留言板可以放一个独立的目录管理，它包括
消息序列和附件两个部分，消息序列只有显示在留言板上的消息，不包括文件信息的内容，
附件另外放。其他方连到这个留言板后，先同步消息序列，如果修改了序列，每次发送一
个diff到被修改的目标上（这个不用纯文本的diff，可以定义一种表示序列什么位置发生
更改的协议），留言板合并其他修改后，返回另一个diff到这个客户端上，形成客户端界
面上的留言更新。

客户端下载留言中的附件，再从服务端下载就行了。这里的附件不一定需要是单个文件，
也可以是个目录，我们也不设置存储上限，因为这本来就是可信任的通讯方之间的，我们
先用到没有空间为止，未来有文件再限制也不影响架构。

每个设备放一个默认的留言板，这样有利于初始化，没有留言板的时候自动创建一个空的
默认留言板。

密码可以是per设备的，用户先在局域网中找到服务，用密码连上设备可以列出留言板，然
后选择一个留言板连入就行了。

### diff协议

消息序列可以是一个带序列号的单条消息的列表，如果消息是一个附件，可以带上特殊标
记说明附件的信息即可。

设备第一次连上的时候先同步一次全集，之后如果主列表发生更新，就针对这个连接加一
条“增删改”的记录，定期同步到客户端即可。

如果设备发生离线，就只能看到离线前的消息（并且同步会发现这个离线，可以进行提示），
如果重新连上，那就会拿到一个新的快照，前述逻辑就会重新成立。
