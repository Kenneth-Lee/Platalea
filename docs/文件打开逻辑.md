# 文件打开逻辑

本应用的文件打开逻辑如下：

* 从其他应用通过本应用打开文件：总是把该文件拷贝到本应用的根目录，如果有重名文件，会询问是否覆盖。

* 本应用内部，分认识的文件和不认识的文件，如果是认识的文件，比如rst，md，gpg，zip等，使用默认的方式打开，比如：

   * .rst和.md文件用rst和md渲染器打开
   * .pdf用.pdf渲染器打开
   * .gpg用解密功能打开
   * .epub文件用epub渲染器打开
   * .zip文件，如果没有前置扩展名，用解压功能打开
   * .pass文件，使用密码查看器打开
  
  如果是不认识的文件，使用内置查看器打开。内置查看器不考虑文件格式，按文本或者二进制方式对原始文件进行查看或者编辑。

* 对于带前缀扩展名的zip文件，使用特殊“解压后查看”的方式处理（具体策略参考README.md），这种文件包括：
  
  * .md.zip
  * .html.zip
  * .rst.zip
  * .pic.zip

* 无论是认识还是不认识的文件，在上下文菜单上总是可以选择如下功能打开：

  * 压缩，即使原文件已经是压缩文件了，仍可以对文件再压缩一次
  * 加密，即使原文件已经是加密文件了，仍可以对文件再加密一次
  * 内置查看器，即使原文件是已经认识的格式了，但仍可以作为一般文件进行原始的编辑

* 对于md或者rst文件，可以在上下文菜单上选择“密码保护”功能，把文件加密为.pass文
  件。这是针对md和rst文件的特殊保护功能。

* 对于.md.pass或者.rst.pass文件，可以选择“直接编辑”功能，解密后编辑对应的.md或者.rst文件，并在完成编辑后重新加密。

# 加解密逻辑

加解密功能尽量减少用户不必要的选择，所以：

* 如果文件已经是gpg格式，单击直接进行解密，而且自动判断加密类型，选择对称或者非对称的方式解密
* 如果要求对文件进行加密，用户也不需选择用对称还是非对称加密，而是列出所有公钥
  以及一个加密密码选项，工具选择了公钥还是密码，决定用非对称还是对称加密
