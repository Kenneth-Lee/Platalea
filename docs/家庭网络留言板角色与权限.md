# 家庭网络留言板：角色与权限

> 身份总览见 [家庭网络共享策略](./家庭网络共享策略.md) §2。本文描述 PC 与手机 HTTP 服务共用的角色模型与 API。

## 1. 核心概念

| 概念 | 说明 |
|------|------|
| **role_class** | `admin` 或 `user`；API 响应字段，表示权限大类 |
| **role_id** | 具体角色 ID，如 `admin`、`guest`、`family`；由密码解析得到 |
| **author_label** | 发言显示名，与 `role_id` 无关，连入者自行填写 |
| **role_ids（板级）** | `meta.json` 中授权可访问该板的 **非 admin** 角色列表 |

**admin** 始终可见并管理全部留言板，不必写入 `role_ids`。**user** 是一类身份，可配置多个具体角色（访客、家人、朋友等），各角色独立密码。

### 1.1 admin 不变式

- admin **始终可见全部留言板**，不检查 `role_ids`。
- `role_ids` **只描述非 admin 角色**；写入时忽略误填的 `"admin"`。
- 新建板省略 `role_ids` 时默认为 `[]`：**仅 admin 可见**。
- **旧板兼容**：`meta.json` **无** `role_ids` 字段 → 对所有 user 角色可见（legacy）。**显式** `role_ids: []` → 仅 admin。

密码校验顺序：**先 admin，再其他 user 角色**（从高到低）。客户端请求头：`X-Network-Service-Password`。

## 2. 服务器配置（PC）

配置文件 `~/.localmanager/config.json`（或 `--config` 指定路径）。

### 2.1 推荐：`roles` 对象

```json
{
  "roles": {
    "admin": {
      "password": "…",
      "label": "管理员",
      "can_create_boards": true,
      "can_manage_boards": true
    },
    "guest": {
      "password": "…",
      "label": "访客",
      "can_create_boards": false,
      "can_manage_boards": false
    },
    "family": {
      "password": "…",
      "label": "家人",
      "can_create_boards": false,
      "can_manage_boards": false
    }
  }
}
```

- 同一密码不得映射多个角色。
- `roles` 必须包含 `admin`。
- 每角色可扩展 `agent` 配置（模型、工具、上下文预算），见下文 §5。

### 2.2 兼容：`guest_password` / `host_password`

未配置 `roles` 时：

| 旧字段 | 映射 |
|--------|------|
| `host_password` | `admin` 角色密码 |
| `guest_password` | `guest` 角色密码 |

## 3. 手机本机服务配置

| 配置项 | 对应角色 |
|--------|----------|
| **设置 → 接入密码（管理员）**（与 FTP 共用） | `admin` |
| **家庭网络 Tab → 管理用户身份** | 自定义 user（朋友、同事等） |

用户身份保存在 `family_network/user_roles.json`，不进入全局配置导出。创建/编辑留言板时在家庭网络 Tab 内勾选可访问的身份。

- **本机应用内**打开留言板：始终为 admin。
- **远程连入本机 HTTP**：管理员使用设置中的接入密码；其他用户使用各自身份密码。

## 4. 留言板元数据

```json
{
  "id": "travel-2025",
  "name": "2025 旅行",
  "revision": 12,
  "role_ids": ["guest", "family"],
  "created_at": 1710000000000
}
```

- 留言板 **显示名不可重名**（创建/重命名时校验）。
- 创建板（admin）：请求体 `name`、可选 `role_ids`；省略时默认 `[]`。

## 5. API 行为摘要

| 接口 | admin | user（且在 role_ids 中） | user（不在列表中） |
|------|-------|--------------------------|-------------------|
| `GET /boards` | 全部板 | 仅授权板 | 不出现 |
| `GET /boards/{id}/messages` | 允许 | 允许 | 404 |
| `POST …/messages` | 允许 | 允许 | 404 |
| `POST /boards` | 允许 | 禁止 | 禁止 |
| `DELETE /boards/{id}` | 允许 | 禁止 | 禁止 |
| `PATCH /boards/{id}` | 允许 | 禁止 | 禁止 |
| `PUT/DELETE …/messages/{id}` | 允许 | 禁止 | 禁止 |

会话响应字段（`GET /boards`、`GET …/messages` 等）：

- `role_id`、`role_class`（`admin` / `user`）
- `role_label`（可选）
- `can_manage`

`GET /boards` 每项对 admin 含 `role_ids`；user 角色不返回或只读。

错误码：`board_name_duplicate`（名称重名）、`unauthorized`（密码错误或缺失）。

## 6. AI Agent 与上下文（PC）

Agent 仅在 PC `platalea` 运行。按**角色**配置 `agent.models`、`agent.tools`；某板被 @ 时校验板 `role_ids` 与角色权限。上下文截断按**被 @ 模型**的 `context_chars` 预算（见策略文档 §5.3）。

## 7. 手机侧：本机 vs 远程

| 模式 | 身份 | 能力 |
|------|------|------|
| **本机留言板**（`isSelf`） | admin | 管理本机全部本地板 |
| **远程连接** | 服务端（或对方手机 HTTP）解析的 `role_id` | 仅该角色允许的操作 |

远程使用 admin 密码时与 PC admin 等价；使用 guest / family 等密码时仅为对应 user。

## 8. 实现状态

| 能力 | PC | 手机 HTTP |
|------|-----|-----------|
| `roles` / 多 user 密码 | ✅ `config.json` | ✅ 家庭网络 Tab 身份管理 |
| 板 `role_ids` 过滤 | ✅ | ✅ |
| `role_class` 响应 | ✅ | ✅ |
| 留言板重名校验 | ✅ | ✅ |
| 按角色 Agent | ✅（部分） | —（不在手机运行） |
