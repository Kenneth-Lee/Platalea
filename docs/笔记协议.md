# 笔记协议

## 介绍

本地管家信息管理而思路是尽量标准化。其他笔记软件的一般思路是设计某种私有格式，
读写，维护，都需要用特定的软件才能访问。而本地管家的笔记尽量就是：我直接帮你改
你的文本文件，这样，就算你不用本地管家，首先你直接就可以看，其次就是，只要你符
合一定的协议，本地管家的功能可以帮你直接维护那部分内容，还能容忍一定的格式变化。

## 本地管家笔记协议

### 数据文件

本地管家使用笔记功能的时候，会先从一个md文件开始同步，这个文件总是找根目录下
的.lm.note.md或者.lm.note.md.gpg文件，如果能找到后者，就使用后者，如果找不到后
者，就使用前者，如果两者都不存在，应用可以创建一个空的.lm.note.md文件。

.lm.note.gpg自动使用应用的默认GnuPG钥匙对进行加密和解密。

每个笔记功能启动的时候通过读取这个文件获得数据，但每个功能都只访问所述功能的对
应章节，忽略其他章节。

比如说，如果我们有一个功能叫“思想火花”，这个功能访问数据文件的时候，总是访问其
中“## 思想火花”这个章节，按里面的内容获得数据，修改完后，同步回笔记也仅仅替换这
个章节，不改变其他部分。

功能处理这部分功能时，应该把整个章节全部读走，作为应用内部数据处理，在处理完成后，
要把数据完整替换整个章节，不应该部分修改其内容，避免其他同步错误。换句话说，功能和
数据文件只有两个独立的交互点：

* 功能启动的时候解释数据文件，把数据文件的对应章节的内容全部作为功能的数据
* 功能退出的时候解释数据文件，用功能本身的数据替换对应章节

中间数据文件再有其他更改不应该和功能发生任何关系。

后面我们介绍不同功能的具体格式

### “思想火花”数据格式

“思想火花”的数据保存在“## 思想火花” 章节中，它的内容包含一组如下格式的记录：

“* [] <思想内容>”

或者

“* [x] <已经完成的思想内容>”

思想内容上加删除线表示该思路可以不需要了，但保留作为记录。

访问思想火花数据的软件读出所有这些记录作为数据，同步回数据文件的时候，根据编辑
好的记录按一样的格式写回到数据文件中，覆盖原来的段落。

“思想火花”章节可以包含一层子章节，每个子章节称为一个“类别”。比如，“### 编程思路”，
就属于“编程思路”类别，里面的内容和在主章节中的内容是一样的。

下面是一个“思想火花”数据格式的例子：
```
## 思想火花

* [] 好好学习
  天天向上
* [x] 记得每天喝牛奶
* [] 饭后要散步

### 编程思路
* [x] 写一个“本地管家”应用
* [] 为“本地管家”增加“思想火花”功能

### 读书笔记

* [] 道可道，非常道
* [x] ~~上德不德，是以有德~~

```
