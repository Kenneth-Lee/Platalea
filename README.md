# Local Manager

这是一个Android应用程序，我把它称为“本地管家”。开发这个软件的原因主要是做很多
Android系统本身不能做，而商业软件常常为了商业目的故意让功能做得有所缺失，我希望
通过这个软件补全这个功能，让手机能当电脑用的地方都可以当电脑用，容易和电脑互通，
而且可以用于一家人互相共享信息。

所以，它是个补全功能，如果你能很容易找到其他软件完成的功能，这里尽量不做。当前
它的功能涉及：加解密，压缩解压缩，文件查看，文件共享，fpt服务器，音乐播放等。每
个功能都不是你在一般软件上看到的那样的（特别是那些你用着不方便的），各位读者可
以在使用中体会。

这个软件完全没有广告，我做出来就是为了解决自己的问题，开放出来也顺便解决你的问
题，没指望用它赚钱，但我不为任何数据破坏负责，请你自担风险。

## 设计思路

作为信息管理软件，本软件（后面统一称为“本地管家”）围绕某个特定的目录来展开功能，
呈现为针对这个目录的文件管理器形式。你可以选择任何应用权限所在范围内的目录作为
工作目录，这个工作目录称为本地管家的“根目录”。

用户可以在这里建立新目录，增，删，改文件，你也可以在其他程序中选择本地管家打开
一个文件，那个文件会被保存到本地管家的根目录中，然后你可以用本地管家的功能来管
理这些文件。（很多人不一定注意到这个功能其实是最重要的：很多软件会控制你只能用
特定的软件打开特定的文件，但如果它可以选择“用其他软件打开”，然后你选择本地管家，
这个文件直接就到你的目录了。）

我们通过控制这个目录的结构，内容，提供所有的功能，同时允许你换一个目录也做一样
的管理，这些功能不强依赖这个目录当前的结构，每次你换用一个新的根目录，我们就按
新目录的内容部署你的功能。

这个理念也体现在回收站的处理上，本地管家的回收站就放在根目录上，所以，你使用不
同的根目录，就会选择不同根目录的回收站。回收站只是一个特定管理的隐藏目录
（.Trash），你甚至可以看到它，操作它，但本地管家不管这些，它就是根据你有没有这
个目录决定怎么操作而已。

更多功能的介绍，请参考下个小节。

## 功能介绍

### 批处理

作为基本的文件管理功能，本地管家提供了一个和其他文件管理不同的批处理操作界面。
你可以在主文件列表中双击把文件或者目录加入“待处理列表”，然后一次对这些文件或者
目录做操作，比如拷贝，移动，压缩，加解密等等。处理目标就是当前路径，所以你要收
集一组文件处理，可以一个个文件，一层层目录找，然后到目标目录做操作就可以了。

文件加入和移出待处理列表可以双击或者从上下文菜单上选择，你还可以对当前目录进行
过滤，然后把过滤好的文件全部加入待处理列表。打开待处理列表也可以过滤，可以把它
们一次移出待处理列表。我们尽量让你在手机上也能方便地管理一组文件，然后你再选择
整体对这些文件做什么处理。

这个功能是后面很多功能的基础。

### ftp服务器

为了让电脑可以传文件或者目录到手机上，我们提供了ftp服务器功能，你启动后它会一直
亮屏，充当一个服务器，你用完退出界面就可以了。这避免了很多其他ftp服务器动不动就
熄屏，然后网络就不通了的问题。

这样会有点浪费电，我建议你插着电用，或者用完就赶紧退出。你也可以在配置菜单上找
到配置退出时间的，如果你怕不记得关，到那里设置一个超时时间也行。

也许你会问为什么不提供更安全的ssh服务？答案是Android的ssh库和PC的很多密码管理协
议不兼容，在Android上使能这个太麻烦。我们通常也只是在局域网上传点东西，如果你怕
不安全，完全可以用GnuPG加密后再传。

### 加解密

作为以信息管理为主的应用，本地管家提供的非常重要的功能是保密，避免在各种不同媒
介之间传递信息时被中间人窃取了信息（比如你要通过某些通讯软件发给文件给别人）。

所以我们提供了丰富的加解密手段：

* GnuPG对称，非对称加解密
  
  * 我们可以在手机上默认创建一对密钥（只会有一对），然后你可以把公钥导出去，提
    供给其他人，其他人用公钥加密，你用私钥解密。这个既可以在手机上完成，也可以
    在PC上装GnuPG完成，GunPG在大部分Linux上是默认安装的，Windows下也有WinGnuPG
    软件。这都是通用的算法。

  * 我们支持导入其他人的公钥，你想和谁通讯，让他把公钥发给你，你导入进来，然后
    你用他的公钥加密，发给他后，他就可以用私钥解密了。
 
  * 如果是对称加密，那就用双方共同认可的密码，这个保密程度会比非对称加密差，但
    用起来就是个密码，简单很多。

* 混淆加密：这个特别设计出来处理大文件的，比如你有个视频有1G，加密这种手机会非
  常慢，但站在防君子不防小人的角度，其实你只要把文件头加密了，让人无法看出文件
  的格式，这也是加密了。这种加密方法如果你拷贝到电脑上就没法解密了，所以我们在
  在项目里提供一个python脚本scripts/quick_obfuscate.py，你可以用它在PC上反混淆
  被手机混淆的内容。

  请注意，混淆算法需要你严格记得密码，如果密码错了，反混淆是永远成功的，只是你
  的文件就彻底毁了而已。

  还要注意混淆是不安全的，比如有些视频，压缩文件，都是一段一段拼接的，就算混淆
  了文件的头，第一段不能用，后面的还是可以用的。还有文本文件，混淆了开头的位置，
  后面的内容都是可以看到的，所以，不要太依赖这个功能做加密。

* zip加密：zip压缩格式也可以设置密码的，本软件提供可以设置密码的zip压缩方式。当
  然，zip加密的可靠性是不如GnuPG加密的，但作为防一般人，有时也够用。

有人记不住很多密码，会用专门密码管理文件记住一些密码，其实没什么必要，因为这本质
不就是把文本文件加密吗？所以，本地管家把上述功能组织成一个方便的密码管理逻辑：

* 你可以把你的密码写到一个rst或者md文件中，然后选择密码保护功能，这会用你的默认
  公钥加密这个文件，但文件名不是.gpg，而是.pass，原文件也会被删除。
* 之后你点击这个文件，我们会自动解密并且把pass文件渲染出来。
* 一般来说，我会建议你先导出本地的公钥，发到电脑上，用公钥加密文件，然后发到手
  机上，之后手机就只查看内容。

有些人可能不知道md/rst文件是什么，它其实就是文本文件，只是如果在文件中加上一些
标记，它显示起来不一样而已。你完全就可以把密码写成文本文件，改名叫xxx.md，然后
就可以用这个功能了。

我用了大部分手段提高可靠性，但这个功能也不能保证完全安全的，下面是一些安全考量，
请使用者注意：

* 私钥是保存在应用沙箱内的，一般情况其他软件不能访问，但root的设备就不同了，所
  以，尽量别在root设备上用。另外，给私钥加上密码，也能加强保护，只是这样你每次
  打开密码文件都需要输入私钥密码。

* md/rst文件查看的内容会保留在WebViewCache中的，我已经在你看完后马上清除Cache了，
  但在你看密码，或者拷贝密码的时候，还是有可能被恶意软件截获的，少装不明来历的
  软件，看完密码尽快退出，可以缩小你被攻击的窗口。

* 你重新生成密码对，原来的密码对就会被覆盖，一旦这样，我们就无法恢复你的密码文
  件了。所以，尽快导出你的公钥，或者把你的密码文件在其他地方保留一份，免得丢失
  数据。

* 即使有了上面的保护，Android程序是运行在Java虚拟机里面的，我们无法避免黑客攻击
  你的Java虚拟机缓存，所以，我们没法做到绝对保密，这只是防止了大部分的攻击。

* 我们这是开源代码，黑客完全可以知道我们用什么办法加密的，随着时间的推移，每种
  加密算法都可能被找到漏洞，所以，不要以为加密了就永远安全了。但通常来说，加密
  这种事情就是看防多少人而已，我们至少能防住大部分的攻击的。PnuPG是国际通用的加
  密工具，一般是不容易攻破的。

### 内置的文件查看器

本地管家提供了很多其他软件不提供的文件查看功能：

* 文本文件查看。你可能觉得这没有什么特别的，但这里最大的区别是这个可以用于查看
  任何文件（包括二进制文件），而且我只看头部的有限内容，所以你无论打开多大的文
  件，都不会挂死。如果你要看全文，我们还提供了“编辑”功能，你自己可以判断文件是
  否真的需要“编辑”，再把全部内容加载进来。

* 这个查看器还可以切换到16进制模式，让你看到它的二进制形式。而且这个16进制查看
  器也可以进入编辑功能，让你一个字节一个字节修改其中的内容。

* 本地管家还提供的rst和md两种markdown的查看支持。如果你不知道它是啥，那就算了。
  但如果你还是懂一点点电脑，知道怎么编辑文件的，我会建议你上网去学习一下，这能
  让你快速写很多格式优美的文本的。

  本地关键支持在md/rst渲染的时候支持内链跳转，图片显示，数学公式，表格，甚至
  mermaid图形。我们还把这个功能和zip压缩集成了，如果你的.zip文件是这样的名字：
  xxx.md.zip或者xxx.rst.zip。那么，我们查看的时候，会自动把内容临时展开，让你可
  以直接看到其中的多个文件，允许索引图像等等。如果zip文件没有加密，那么加压的结
  果作为cache使用，只要这个压缩文件没有更新过，下次使用直接使用cache的数据，不
  需要再次解压。如果压缩文件是加密的，那么Cache在使用后会自动删除。

* rst.zip和md.zip功能对html也有效，如果包含html的目录打包成.html.zip，查看器可
  以打开其中的index.html，并使用 cache 中的 CSS/JS/图片等资源。对于加密压缩的处理
  逻辑也和md.zip或者rst.zip类似。

### 音乐播放器

音乐播放器够多了，我这个就解决本地播放的问题：你可以把待处理列表中的mp3或者ogg
转成一个playlist，然后我们就播这个列表。你可以创建很多的列表，每个列表会独立记
录当前的播放位置。这样，这段时间你听书，下段时间听白噪声，换个时间听流行乐，你
就建立不同的列表，根据需要切换就行了。

### 数据共享功能

我们有时希望一家人可以共享一些公共的信息，比如某次一家人去旅行，不同的人整理了
不同的信息，都提供给管家，大家都能看到。

这种信息最适合的就是用git来管理，所以我们提供了一个基础的构架，就是可以为管家配
套一个公共的git库，管家每次都可以pull到本地(.sysgit目录）来，如果本地更新了，也
可以push会公共的git库上，这样我们就有一个公共的贡献信息了。当然这个我们不指望很
大，能分享一些文本就行。

这个目录我们不打算让人为去维护，而是让程序去维护，通常某个功能提供一个界面让用
户修改一些文件，修改前用户pull内容，修改后push回去，整个过程用户是不感知这个公
共目录的存在的。

这种方法，你很容易通过电脑直接维护这个库，控制所有的行为。如果出现push/pull冲突，
让本地手机直接清空一次.sysgit就行，只会丢失最后一次维护的数据。唯一的问题是这种
方法会比较慢。

现在这个功能主要可以共享两种数据：

* gpg的公钥，请在公钥管理找“公钥分享”

* 简单的文件直接共享，这个可以在文件的上下文菜单中发送，在主菜单中找“git文件共
  享”管理共享的文件。

对一般用户（非程序员用户），可能不能接受这个功能的速度，因为它就不是个网盘。所
以，如果不是程序员，就别用这个功能吧，它是个包含历史管理的工作仓，未来我们是希
望可以在后台运行一个AI管家去管理其中的内容的。

## 开发与构建

### 环境要求

- **JDK 17+**（推荐 21）
- **Android SDK**（含 Platform 34、Build-Tools 34.0.0）
- **Gradle**（已安装时可直接用 `gradle`；否则使用项目自带的 `./gradlew`，需能下载 Gradle 发行包）

### 必须完成的配置

1. **设置 Android SDK 路径**（若未设置）  
   例如：
   ```bash
   export ANDROID_HOME=$HOME/Android/Sdk
   export PATH=$PATH:$ANDROID_HOME/platform-tools
   ```
2. **接受 SDK 许可**（未接受时构建会报 “licences have not been accepted”）  
   ```bash
   sdkmanager --licenses
   ```
   按提示输入 `y` 接受全部许可。

### 构建与安装

```bash
# 使用系统 Gradle（你已安装时）
gradle assembleDebug

# 或使用项目 Wrapper（会下载 Gradle，需网络）
./gradlew assembleDebug
```

安装到已连接设备或模拟器：

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

### 若构建失败

- **“Could not determine the dependencies” / “licences have not been accepted”**  
  执行 `sdkmanager --licenses` 并接受所有许可后重试。
- **“sdkmanager: command not found”**  
  安装 Android 命令行工具（如通过 Android Studio SDK Manager 安装 “Android SDK Command-line Tools”），或确保 `$ANDROID_HOME/cmdline-tools/latest/bin` 在 `PATH` 中。
- **Gradle 下载慢或超时**  
  可配置代理或使用国内镜像（如修改 `gradle/wrapper/gradle-wrapper.properties` 中的 `distributionUrl` 为阿里云等镜像），或使用已安装的系统 `gradle` 进行构建。
