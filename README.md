# Local Manager

这是一个 Android 应用程序，主要用作 Android 手机的信息管理。它首先呈现为一个文件浏览器，可以查看手机上有哪些文件，并支持**打开、重命名、移动、拷贝、删除**文件。另外，它支持 **GnuPG 兼容**的加密和解密（对称加密）。内置**文本**和**十六进制**查看/编辑功能，可快速以文本或十六进制方式查看并编辑文件内容。支持**回收站**及从回收站**恢复**。未来还会增加更多功能。

## 功能概览

- **文件浏览**：通过主菜单「更换根目录」授权后，以树形方式浏览目录、进入子目录、返回上级。
- **文件维护**：点击 FAB「新建」可选 **新建文件** 或 **新建文件夹**；长按项可 **重命名**、**删除**（带确认，可移入回收站或彻底删除）、**加入待处理列表** 后 **拷贝/移动**。可通过主菜单 **显示/隐藏过滤条件**；显示时，列表下方的正则过滤框可输入正则表达式过滤当前目录（留空显示全部）。
- **回收站**：删除文件/目录时可选移入根目录下的 `.Trash` 回收站；进入回收站后可 **恢复** 到根目录，或 **清空回收站**。长按项可选择「恢复」。
- **查看文件**：见下方说明。
- **GnuPG**：对 `.gpg` 文件尝试解密后查看（对称加密需在应用内输入密码）；支持密钥管理（导入/导出、新建密钥等）。

### 查看文件

- **点击文件**：默认使用系统应用打开（系统会弹出“用以下方式打开”选择器）；若没有应用能处理该类型，则自动改用内置查看器。
- **长按文件**：弹出上下文菜单，可选择 **用内置查看器打开**、**加入待处理列表**、**重命名** 或 **删除**。
- **内置查看器**：
  - 支持 **文本** 与 **十六进制** 两种视图切换；十六进制视图每行左侧为 hex、右侧为 ASCII（不可见字符显示为点）。
  - 默认显示文件前 **4096 字节**（可在配置中修改，范围 1024～10M）；文本模式下按 UTF-8 解码，若遇多字节编码会显示到最后一个完整有效字符。
  - 若内容未完整显示，会提示「内容未完整显示，还有 X 未显示」，X 以 K/M/G/T 等易读单位表示剩余大小。
  - **十六进制编辑**：在十六进制视图下可进入编辑模式，逐字节修改；每格可完整显示两位十六进制并可横向滚动查看整行。
  - 使用异步加载，未加载完成也可直接返回退出。
- **拷贝/移动**：长按文件或目录可 **加入待处理列表**；在任意目录下通过顶部菜单可 **拷贝到本处** 或 **移动到本处**，并可打开 **待处理列表** 编辑（移除项）。

### 配置

主菜单 → **配置** 中可设置：

- **显示/隐藏过滤条件**：控制是否显示列表下方的正则过滤框（主菜单中亦可快速切换）。
- **不显示 . 开头的文件**：开启后隐藏 `.gitignore`、`.Trash` 等以点开头的文件和目录。
- **内部查看器预览长度（字节）**：设置内置查看器单次加载的字节数，默认 4096，范围 1024～10M。

## 开发与构建

### 环境要求

- **JDK 17+**（推荐 21）
- **Android SDK**（含 Platform 34、Build-Tools 34.0.0）
- **Gradle**（已安装时可直接用 `gradle`；否则使用项目自带的 `./gradlew`，需能下载 Gradle 发行包）

### 必须完成的配置

1. **设置 Android SDK 路径**（若未设置）  
   例如：
   ```bash
   export ANDROID_HOME=$HOME/Android/Sdk
   export PATH=$PATH:$ANDROID_HOME/platform-tools
   ```
2. **接受 SDK 许可**（未接受时构建会报 “licences have not been accepted”）  
   ```bash
   sdkmanager --licenses
   ```
   按提示输入 `y` 接受全部许可。

### 构建与安装

```bash
# 使用系统 Gradle（你已安装时）
gradle assembleDebug

# 或使用项目 Wrapper（会下载 Gradle，需网络）
./gradlew assembleDebug
```

安装到已连接设备或模拟器：

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

### 若构建失败

- **“Could not determine the dependencies” / “licences have not been accepted”**  
  执行 `sdkmanager --licenses` 并接受所有许可后重试。
- **“sdkmanager: command not found”**  
  安装 Android 命令行工具（如通过 Android Studio SDK Manager 安装 “Android SDK Command-line Tools”），或确保 `$ANDROID_HOME/cmdline-tools/latest/bin` 在 `PATH` 中。
- **Gradle 下载慢或超时**  
  可配置代理或使用国内镜像（如修改 `gradle/wrapper/gradle-wrapper.properties` 中的 `distributionUrl` 为阿里云等镜像），或使用已安装的系统 `gradle` 进行构建。
