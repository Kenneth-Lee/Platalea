# Platalea (Local Manager)

这是一个Android应用程序，我把它称为“本地管家”，表示它主要做一些本地文件或者其他
信息管理的功能。但这个名字太普通了，上传到一些应用市场的时候，很容易重名，所以
我又把应用的名字改为Platalea，这是琵鹭的学名，没什么特别的意思，一方面它是一种
广东常见的鸟，应用是在广东开发的，另一方面名字足够“不一样”，如此而已。

开发这个软件的原因主要是用来做很多基本系统本身不提供，而商业软件常常为了商业目
的故意让功能做得有所缺失，的特性。我希望通过这个软件补全这些功能，让手机能当电
脑用的地方都可以当电脑用，容易和电脑互通，而且可以用于一家人互相共享信息。

这里说“一家人”，主要是这个应用只用于一个团队（比如共享目录就只能设置一个，不指
望你切换），所以，只要你是用于一个团队就行，是不是一家人不重要。而且，你不需要
复杂的共享功能，那就更不用管了。

总的来说，它是个功能补全。如果你能很容易找到其他软件完成的功能，这里尽量不做。
当前它的功能主要涉及：加解密，压缩解压缩，文件查看，文件共享，fpt服务器，音乐播
放等。每个功能都不是你在一般软件上看到的那样的（因为主要是补全它们的缺失），各
位读者可以在使用中体会。

这个软件完全没有广告，我做出来就是为了解决自己的问题，开放出来也顺便解决你的问
题，没指望用它赚钱，但我不为任何数据破坏负责，请你自担风险。

本项目也上传到华为市场了，但只有国际版本，没有国内版本。

最新一个发布版本在这里：[LocalManager-V0.24](https://atomgit.com/Kenneth-Lee-2025/local_manager/releases/download/V0.24/app-release.apk)

## 设计思路

作为信息管理软件，本软件（后面统一称为“本地管家”）围绕某个特定的目录来展开功能，
呈现为针对这个目录的文件管理器形式。你可以选择任何应用权限所在范围内的目录作为
工作目录，这个工作目录称为本地管家的“根目录”。

根目录的存在让我们完成很多功能的时候不需要询问你复杂的信息，因为我们都可以以根
目录和当前浏览的目录作为目标目录。比如回收站总是根目录下的.Trash目录，笔记总是
根目录下的.lm.note.md，不需要问你一堆复杂的问题。

用户可以在根目录增、删、改文件和目录，可以查看目录的内容，本地管家有一组内置的
查看器，让你可以用本地管家本身阅读这些文件，比如：

* Markdown文件：.md和.rst文件都支持。这两者都是文本文件，但可以通过一些简单的标
  识，显示出来呈现为一个带丰富的格式的文档的样子。这用来在网上做信息共享非常方
  便。本地管家对md/rst格式做了尽量多的支持，比如本地文件之间的跳转，数学公式，
  图表，代码高亮等等，都是支持的。

* .epub：这是电子书的标准开放格式。本地管家不是一个阅读软件，但临时看到一个epub
  文件，直接当本书看还是可以的。比一般商用阅读器少了各种出于推广目的骚扰你的内
  容。epub阅读器可以使用星际译王的词典。除了.epub，默认txt和llm文件也用.epub的
  方式查看。llm文件是我另一个项目，vim_with_llm的文件，也是一种文本文件，只是用
  更接近AI沟通的方式显示而已。

  本地管家的epub阅读器支持离线语音朗读，注意不同语言需要选择不同的朗读语音。

* .pdf：本地管家显示pdf的效果一般，但如果你需要快速看一下pdf的内容，这也是可以
  用的。

其他本地管家不直接支持的功能，都可以选择用其他应用打开。另外，本地管家提供了一
个万能的内置查看器。如果不是已知的格式，用这个内置查看器打开，都是当作文本文件
显示文件前面的一些内容，你也可以在查看器中切换到16进制模式。这带来两个好处：第
一，不管是什么文件，你都可以放心打开，因为我们只读很少的一部分，这不会很消耗资
源。对于专业人士来说，看看文件头，已经可以知道很多信息了。第二，如果你看了头部，
觉得需要看或者修改更多的信息，你完全可以进入编辑模式，以文本或者16进制的方式编
辑里面的内容。

对于已知的格式，内部查看器可以用上下文菜单打开。

除了这些基本的查看功能外，本地管家充分利用了zip的优势。首先，本地管家支持zip压
缩和解压缩，也支持zip压缩文件加密。而我们分享的很多文件，比如html，md，rst文件，
常常都是包含多个文件的。你把这些文件打包成一个zip文件，本地管家可以直接针对这个
目录进行浏览。比如说，你有几个md文件，还有它引用的一些图片在一个目录中，你把这
个目录压缩一下，叫做xxx.md.zip，那么用本地管家可以直接打开（自动查找里面的
index.md或者为你生成一个），如果你下载了一个又有图片，又有css的网页，你也把这个
目录整个打包（注意网页本身必须在根目录上），然后你整个zip文件发到手机上，也可以
直接用本地管家直接打开。

正如我们前面说的，本地管家主要做的是其他软件做不了的功能（或者不方便的功能），
所以，一般有解决方案的，这里就不做了。无论本地管家是否支持，你都可以随时选择“用
其他软件打开”，这样我们可以用其他软件的能力来打开这个文件。

反过来，其他应用也可以利用本地管家的能力。如果你用其他应用的时候，选择让本地管
家打开它的文件，本地管家会把那个文件拷贝到自己的根目录，然后你喜欢怎么看就怎么
看了。很多人不一定注意到这个功能其实是最重要的：很多软件会控制你只能用特定的软
件打开特定的文件，但如果它可以选择“用其他应用打开”，然后你选择本地管家，这个文
件直接就到你的目录了。比如别人通过通讯给你发了个附件，很多商用软件只允许特定的
应用打开那个附件，你死活都保存不到本地，这种时候本地管家就可以轻易解决你的问题。

如果还不行，本地管家也提供ftp和私有服务器（称为“家庭网络”），允许你在PC和手机之
间直接传输文件。

对于团队或者家庭，本地管家还允许配置一个git库，让整个团队或者家庭成员一起用一个
git库来共同维护共同的数据，这也可以让你直接拿到一些文件。

更多功能的介绍，请参考下个小节。

## 功能介绍

### 批处理

作为基本的文件管理功能，本地管家提供了一个和其他文件管理不同的批处理操作界面。
你可以在主文件列表中双击把文件或者目录加入“待处理列表”，然后一次对这些文件或者
目录做操作，比如拷贝，移动，压缩，加解密等等。处理目标就是当前路径，所以你要收
集一组文件进行批处理，可以一个个文件，一层层目录找，找完所有的文件后，到目标目
录做操作就可以了。

文件加入和移出待处理列表可以双击或者从上下文菜单上选择，你还可以对当前目录进行
过滤或者进行查找，然后把过滤好的文件或者查找出来的文件全部加入待处理列表。打开
待处理列表也可以过滤，可以把它们一次移出待处理列表。我们尽量让你在手机上也能方
便地管理一组文件，然后你再选择整体对这些文件做什么处理。

这个功能是很多其他功能的基础，因为其他功能需要多个操作目标的时候，都是基于待处
理列表来的。

### ftp服务器

为了让电脑可以传文件或者目录到手机上，我们提供了ftp服务器功能，你启动后它会一直
亮屏，充当一个服务器，你用完退出界面就可以了。这避免了很多其他ftp服务器动不动就
熄屏，然后网络就不通了的问题。

这样会有点浪费电，我建议你插着电用，或者用完就赶紧退出。你也可以在配置菜单上找
到配置退出时间的。所以，如果你怕不记得关，到那里设置一个超时时间也行。

也许你会问为什么不提供更安全的ssh服务？答案是Android的ssh库和PC的很多密码管理协
议不兼容，在Android上使能这个太麻烦。我们通常也只是在局域网上传点东西，如果你怕
不安全，完全可以用GnuPG加密后再传。

### 加解密

作为以信息管理为主的应用，本地管家提供的非常重要的功能是保密，避免在各种不同媒
介之间传递信息时被中间人窃取了信息（比如你要通过某些通讯软件发给文件给别人）。

所以我们提供了丰富的加解密手段：

* GnuPG对称，非对称加解密
  
  * 我们可以在手机上默认创建一对密钥（只会有一对），然后你可以把公钥导出去，提
    供给其他人，其他人用公钥加密，你用私钥解密。这个既可以在手机上完成，也可以
    在PC上装GnuPG完成，GunPG在大部分Linux上是默认安装的，Windows下也有WinGnuPG
    软件。这都是通用的算法。

  * 我们支持导入其他人的公钥，你想和谁通讯，让他把公钥发给你，你导入进来，然后
    你用他的公钥加密，发给他后，他就可以用私钥解密了。
 
  * 如果是对称加密，那就用双方共同认可的密码，这个保密程度会比非对称加密差，但
    用起来就是个密码，简单很多。

* 混淆加密：这个特别设计出来处理大文件的，比如你有个视频有1G，加密这种手机会非
  常慢，但站在防君子不防小人的角度，其实你只要把文件头加密了，让人无法看出文件
  的格式，这也是加密了。这种加密方法如果你拷贝到电脑上就没法解密了，所以我们在
  在项目里提供 PC 命令 `platalea file obfuscate` / `platalea file deobfuscate`
  （亦可用兼容脚本`scripts/quick_obfuscate.py`），你可以在 PC 上反混淆被手机混
  淆的内容。

  请注意，混淆功能需要你严格记得密码，如果密码错了，反混淆是永远成功的，只是你
  的文件就彻底毁了而已。

  还要注意混淆是不安全的，比如有些视频，压缩文件，都是一段一段拼接的，就算混淆
  了文件的头，第一段不能用，后面的还是可以用的。还有文本文件，混淆了开头的位置，
  后面的内容都是可以看到的，所以，不要太依赖这个功能做加密。

* zip加密：zip压缩格式也可以设置密码的，本软件提供可以设置密码的zip压缩方式。当
  然，zip加密的可靠性是不如GnuPG加密的，但作为防一般人，有时也够用了。

  对于那些.md.zip，.rst.zip, .html.zip, .pic.zip文件，我们在查看的时候会把内容
  解压到内部缓存中。如果zip文件没有加密，这些缓存会保留，直到zip文件本身被更新
  了。但如果zip文件是加密的，那么文件看完后，缓存会被自动清除，这也一定程度上提
  高安全性。

有人记不住很多密码，会用专门密码管理软件记住一些密码，其实没什么必要，因为这本
质不就是把文本文件加密吗？所以，本地管家把上述功能组织成一个方便的密码管理逻辑：

* 你可以把你的密码写到一个txt，rst或者md文件中，然后选择密码保护功能，这会用你
  的默认公钥加密这个文件，但文件名不是.gpg，而是.pass，原文件也会被删除。
* 之后你点击这个文件，我们会自动解密并且把pass文件渲染出来。
* 一般来说，我会建议你先导出本地的公钥，发到电脑上，用公钥加密文件，然后发到手
  机上，之后手机就只查看内容。（但如果你实在有需要，我们也允许你直接编辑.pass文
  件的，只要在文件的上下文菜单中选择“直接编辑”就行）

有些人可能不知道md/rst文件是什么，它其实就是文本文件，只是如果在文件中加上一些
标记，它显示起来不一样而已。如果你实在懒得管它是什么，就用.txt来加密吧。

我用了大部分手段提高可靠性，但这个功能也不能保证完全安全的，下面是一些安全考量，
请使用者注意：

* 私钥是保存在应用沙箱内的，一般情况其他软件不能访问，但root的设备就不同了，所
  以，尽量别在root设备上用。另外，给私钥加上密码，也能加强保护，只是这样你每次
  打开密码文件都需要输入私钥密码。

* md/rst文件查看的内容会保留在WebViewCache中的，我已经在你看完后马上清除Cache了，
  但在你看密码，或者拷贝密码的时候，还是有可能被恶意软件截获的，少装不明来历的
  软件，看完密码尽快退出，可以缩小你被攻击的窗口。

* 你重新生成密码对，原来的密码对就会被覆盖，一旦这样，我们就无法恢复你的密码文
  件了。所以，尽快导出你的配置（导出的配置中包括你的所有钥匙），或者把你的密码
  文件在其他地方保留一份，免得丢失数据。

* 默认的情况下，Android是通过输入法输入内容的，输入法本身会记忆常用字词，中文输
  入法还会把你的输入呈现出来再输入到对话框中，所以我们这里在输入密码的时候，用
  的是自己的键盘，避免被你安装的输入法截流。但这样会导致你无法输入中文密码，我
  们是建议你不要使用中文密码，但如果你确实需要，可以先在其他地方写好，拷贝到粘
  贴板，然后在这里拷贝进来，这是有风险的，我们这里需要先告诉你。

* 即使有了上面的保护，Android程序是运行在Java虚拟机里面的，我们无法避免黑客攻击
  你的Java虚拟机缓存，所以，我们没法做到绝对保密，这只是防止了大部分的攻击。

* 我们这是开源代码，黑客完全可以知道我们用什么办法加密的，随着时间的推移，每种
  加密算法都可能被找到漏洞，所以，不要以为加密了就永远安全了。但通常来说，加密
  这种事情就是看防多少人而已，我们至少能防住大部分的攻击的。GnuPG是国际通用的加
  密工具，一般不容易攻破。

* 查看加密文件的时候，我没有禁止截屏和拷贝，因为有人可能就是要截屏和拷贝的。所
  以如果有恶意软件有这个权限，也可能产生密码泄漏。

### 快速文本加密

有时我们会通过即时通讯软件发送一些消息，但你不想发送文件那么复杂，我们提供一种
更简单的加密方式，你和另一方共享同一套密钥（可以通过备份你的配置，然后直接导入
到对方的机器上），然后打开本地管家的“快密”功能，把文本拷贝进去，加密，然后把加
密的文本拷贝到即时通讯软件上，发送给对方，对方就可以用同一个功能，把内容解密出
来。

这个功能和“密码保护”功能类似，也是直接使用你的本地密钥。和密码保护功能唯一的区
别是不用你创建文件。

### 更多的压缩格式

本地管家还支持7z和rar格式，不过后者是商业格式，所以只能支持rar4的解压，不能压缩，
我们支持这两种新格式主要是为了避免有时收到别人发的这种文件不好处理而已，聊胜于
无，一般不建议使用。zip格式在很多平台上都是有硬件加速的，用zip会更推荐。

### 内置的文件查看器

如前所述，本地管家主要给你提供信息分享的功能，所以本身提供了一组单个的，或者基
于zip文件的查看功能。这里把所有这些查看功能做一个完整的列表：

* 内置查看器：这个是通用的，可以看文本或者16进制，可以编辑。
* Markdown渲染：这个支持rst和md，支持内链跳转，外链调用浏览器，图片显示，数学公
  式，表格，mermaid图形，等等。本地管家的渲染器支持部分Typora等平台才支持的扩展
  语法，比如上标下标除了可以用sup/sub语法，也支持^和~语法。
* epub
* pdf

对于zip打包格式，我们支持：

* .md.zip，自动查看其中的index.md或者README.md，如果两者都不存在，会自动生成一
  个index.md，其中包含到所有其他md文件的列表。
* .rst.zip，自动查看其中的index.rst或者README.rst
* .html.zip，自动查看其中的index.html，如果没有index.html，会查看第一个找到的
  html文件。
* .pci.zip，自动查看其中的图片
* .llm.zip，自动用epub一样的方法查看其中的.txt或者.llm文件，如果文件中有.index
  文件，把该文件作为章节的定义文件，具体格式请参考 docs/文本压缩文件index协议.md

所有的zip打包格式我们都支持缓存，已经解压过的信息，如果没有加密，我们会保留解压
的信息。对于图片，由于文件可能很大，我们是按“懒惰算法”来解压的（就是你看什么图
片我们再解压什么图片），所以退出查看的时候，加密的图片我们会询问你是否保留缓存
（这会有一点风险，但保证速度）。在配置中你可以找到缓存管理，把不需要的缓存清理
掉。

缓存都是全局的，不会因为你切换根目录造成影响。

### 音乐播放器

音乐播放器够多了，我这个就解决本地播放的问题：你可以把待处理列表中的mp3或者ogg
转成一个playlist，然后我们就播这个列表。你可以创建很多的列表，每个列表会独立记
录当前的播放位置（这是和一般播放器最大的不同）。这样，这段时间你听书，下段时间
听白噪声，换个时间听流行乐，你就建立不同的列表，根据需要切换就行了。

我们允许给列表设置书签，你听到某个精彩的地方可以记录下来，以后可以从那里重听。
也允许随时按不同方式重新排序列表序列。这两个功能说起来互相冲突，但就看你如何组
合使用而已。

音乐列表不和根目录绑定，你可以切换到音乐目录，创建一批列表，然后切换回你的工作
目录，这是没有问题的。

列表的内容之间可以互相拷贝和移动，你可以用这种方法收集不同的列表，使用不同的顺
序等等。列表自动消除重复文件，剔除已经删除文件。

上面的列表其实都是文件的一个指针，我们还提供一种“目录列表”，这种列表通过指定目
录指定，这种目录不是指针，就表示那个目录的内容，这种列表如果你删除音乐，就真的
把文件一起删除了。这种主要是为了让你下载了一个目录很多音乐的情况下，听的时候觉
得那个不顺眼，顺手删掉，不再保留。

应用提供了一个快速入口，你可以在配置窗口中为播放器创建独立的桌面入口，从播放器
入口可以直接进入播放器，操作完马上退出，不需要额外的步骤。


### 数据共享功能

我们有时希望一家人可以共享一些公共的信息，比如某次一家人去旅行，不同的人整理了
不同的信息，都提供给管家，大家都能看到。

这种信息最适合的就是用git来管理，所以我们提供了一个基础的构架，就是可以为管家配
套一个公共的git库，管家每次都可以pull到本地(.sysgit目录）来，如果本地更新了，也
可以push会公共的git库上，这样我们就有一个公共的贡献信息了。当然这个我们不指望很
大，能分享一些文本就行。

这个目录我们不打算让人为去维护，而是让程序去维护，通常某个功能提供一个界面让用
户修改一些文件，修改前用户pull内容，修改后push回去，整个过程用户是不感知这个公
共目录的存在的。

这种方法，你很容易通过电脑直接维护这个库，控制所有的行为。如果出现push/pull冲突，
让本地手机直接清空一次.sysgit就行，只会丢失最后一次维护的数据。唯一的问题是这种
方法会比较慢。

现在这个功能主要可以共享两种数据：

* gpg的公钥，请在公钥管理找“公钥分享”

* 简单的文件直接共享，这个可以在文件的上下文菜单中发送，在主菜单中找“git文件共
  享”管理共享的文件。

对一般用户（非程序员用户），可能不能接受这个功能的速度，因为它就不是个网盘。所
以，如果不是程序员，就别用这个功能吧，它是个包含历史管理的工作仓，未来我们是希
望可以在后台运行一个AI管家去管理其中的内容的。

至于网盘，我这里没打算集成，因为你完全可以用网盘共享某个目录，让本地管家直接给
你管理那个目录。

### 家庭分享功能

家庭分享功能通过mDNS协议允许装了本应用的程序向对方的留言板留言或者留文件，多个
用户连到同一个设备的留言板留言，可以形成一个群聊。

这个功能带来最基本的好处是你不需要启动ftp服务器，就可以在局域网下，在多个手机之
间互相共享文件，或者共同就一个问题留下一个讨论线索。

它带了的第二个好处是，在本工程的 `pc_tools` 目录下我们提供了 **`platalea`** 命
令行工具，你可以在你的家庭PC（Linux，Windows，Mac都行）上启动本地服务，这样那台
PC 就变成一个家庭的公告板，你可以随时上去看家里有什么“新的公告，或者新的共享文
件”。

这个功能和ftp功能一样，打开的时候开启服务，离开页面就停止服务，开启页面的时候会
自动防止休眠。

### git

git功能是一个更通用的git访问功能，它共享前一个功能的配置，允许你下载同一个git网
站的其他项目。这个就是为了把git功能集成进来，让你可以在手机上临时下载一些数据来
看而已。

git功能主要是让你专注用一个git服务器来下载项目，省得像PC中一样要配置一堆东西才
能访问内容。所以很多东西尽量用一个配置参数，而不需要你配置很多内容。但有时你也
确实有可能需要临时下载某个项目，这种情况，作为应急，你也可以临时配置另一个服务
器，但不要“应用配置”，这样不会下载共享功能中的基础项目，然后你可以临时用git功能
下载那个项目，然后再恢复原来的配置。

### 快速笔记

有时我们需要快速记录一些信息，这可以打开一个文本文件来创建，但这需要好几个步骤，
不方便。使用专门的笔记软件呢，这东西的格式又是私有的，导出又是麻烦事。

我们统一解决这个问题：打开快速笔记功能，可以记录一些思想火花，记录完了，这个信
息会被写到管家根目录的.lm.note.md文件中。这是个md文件（也就是文本文件），换个平
台你也可以直接看。而且我们不修改这个文件的其他内容，我们只修改“## 思想火花”这个
章节的内容。其他部分你写什么笔记，我们这里都不管。

这个笔记也提供加密功能，如果你用私钥加密.lm.note.md，产生一个.lm.note.md.gpg文
件，那么笔记功能优先使用这个加密的版本。

和播放器一样，你可以在配置中为快速笔记功能创建一个快捷方式，在桌面可以快速进入
这个功能。

### 读书笔记

读书笔记功能和快速笔记功能类似，但它主要用于关联epub阅读器的书签功能，每个书签
都可以加一些笔记功能。这些笔记和快速笔记一样，统一记录到.lm.note.md文件中，放到
另一个独立的章节（“读书笔记”）中。

读书笔记不和书的文件绑定，而是和书名绑定，同一个书名记录同一套笔记。这有助于你
用不同格式和文件看同一本书记录笔记，但这样书签的位置可能不对。我们的策略是不对
就不对，这就是个参考，你可以认为这是一个提示而已，但我们也允许你在阅读到对应的
位置时更新书签的位置。

### 配置导入导出

这个应用的配置挺复杂的，所以我们支持配置的导入导出，这个导出会把你的私钥也一并
导出，所以这个导出文件最好立即做个加密再发到电脑上保存，免得出问题。

导入导出包括默认密钥和播放列表，这会产生覆盖，导入的时候要要注意备份原来的配置。

### 启动密码

如果你的私钥是有密码的，可以在配置中设置“启动解锁私钥”，这会提供一个简单的安全
功能：应用启动的时候会询问私钥密码。如果没有密码就进不了应用，而且此后你再调用
私钥，就不再询问密码了。

### epub阅读器和字典

epub阅读器是本地管家的核心阅读器，它不但可以阅读epub文件，也可以阅读.llm.zip，独
立的txt文件等。这些都是当作电子书籍来看待的，所以我们为这个阅读器内置了字典。这
和你独立安装的字典软件不同，我们的字典和正文是在一个屏幕上显示的，以便你阅读英
文的时候需要频繁重新打开字典界面，只要你选中一个单词并拷贝它，字典界面就会更新。

我们使用的字典是星际译王的格式，你可以在网上找到免费的资源，下载文件后，在文件
浏览器上找到对应文件，在上下文菜单上可以选择导入。我们可以支持最多3个字典，但每
个时刻你只能激活一本字典。

### 浮动按钮

epub和pdf阅读器中我们都集成了浮动按钮，用于切换下一页。屏幕左右点击也可以完成一
样的功能，这主要用于单手握持大屏或者pad设备用的，你可以根据哪只手握持，切换浮动
按钮的位置，获得最好的阅读体验。这个功能可以在配置中关闭。

浮动按钮可以双击切换全屏模式。

### 历史记录

所有正经的阅读功能，包括阅读epub，用第三方软件打开文件，阅读带前缀的zip文件，播
放某个音乐列表等，都会产生历史记录。你可以直接在“最近”功能中直接重新继续原来的
阅读，大部分阅读可以保留原来的状态，从你离开的位置继续。

但压缩，解压缩，用内置查看器打开文件这些操作，看作是文件目录本身的处理，不产生
历史记录，以免对你产生干扰。

## pc_tools

pc_tools目录中我们提供了一个用来配合手机使用的pc_tools，它是一个python程序，可
以用pip install -e .的方式安装。这个工具在Mac/Windows/Linux上都是可以工作的。安
装后，它有一个统一的命令入口：platalea。

platalea命令可以混淆/反混淆，加密/解密，提供家庭共享留言板或者访问手机的家庭共
享留言板等，可以直接导入手机侧的配置。具体原理和用法，
请参考[pc_tools/README.md](pc_tools/README.md)。

platalea的家庭留言板功能支持AI引擎，你可以在留言板中呼叫本地AI帮你分析报告，总
结留言内容等。AI会在留言板中直接回应你。

## 开发与构建

### 环境要求

- **JDK 17+**（推荐 21）
- **Android SDK**（含 Platform 34、Build-Tools 34.0.0）
- **Gradle**（已安装时可直接用 `gradle`；否则使用项目自带的 `./gradlew`，需能下载 Gradle 发行包）

### 必须完成的配置

1. **设置 Android SDK 路径**（若未设置）  
   例如：
   ```bash
   export ANDROID_HOME=$HOME/Android/Sdk
   export PATH=$PATH:$ANDROID_HOME/platform-tools
   ```
2. **接受 SDK 许可**（未接受时构建会报 “licences have not been accepted”）  
   ```bash
   sdkmanager --licenses
   ```
   按提示输入 `y` 接受全部许可。

### 构建与安装

```bash
# 使用系统 Gradle（你已安装时）
gradle assembleDebug

# 或使用项目 Wrapper（会下载 Gradle，需网络）
./gradlew assembleDebug
```

安装到已连接设备或模拟器：

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

### 若构建失败

- **“Could not determine the dependencies” / “licences have not been accepted”**  
  执行 `sdkmanager --licenses` 并接受所有许可后重试。
- **“sdkmanager: command not found”**  
  安装 Android 命令行工具（如通过 Android Studio SDK Manager 安装 “Android SDK Command-line Tools”），或确保 `$ANDROID_HOME/cmdline-tools/latest/bin` 在 `PATH` 中。
- **Gradle 下载慢或超时**  
  可配置代理或使用国内镜像（如修改 `gradle/wrapper/gradle-wrapper.properties` 中的 `distributionUrl` 为阿里云等镜像），或使用已安装的系统 `gradle` 进行构建。
