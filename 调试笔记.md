## 查看文件与拷贝/移动 测试要点

- **点击文件**：应用应尝试用系统应用打开；若无应用处理则进入内置查看器。
- **长按文件**：应出现「用内置查看器打开」「加入待处理列表」「重命名」「取消」。
- **内置查看器**：文本/十六进制切换正常；十六进制每行左侧为 hex、右侧为 ASCII，不可见字符为点；大文件只显示前 4096 字节；加载中按返回可立即退出。
- **待处理列表**：长按若干文件/目录选「加入待处理列表」后，顶部出现待处理图标；点击可选「拷贝到本处」「移动到本处」「待处理列表」；在目标目录选「拷贝到本处」或「移动到本处」后列表刷新且 Toast 提示；打开「待处理列表」可逐项移除。

## GnuPG 对称加密格式（与 Linux 一致）

**对称加解密已改为使用 [PGPainless](https://github.com/pgpainless/pgpainless) 库**，与 GnuPG/OpenPGP 标准兼容，避免自实现与标准不一致导致同密码无法解密的问题。公钥加密/私钥解密仍使用 Bouncy Castle。

### 依赖

- `org.pgpainless:pgpainless-core`（对称加解密）
- Bouncy Castle：bcpg、bcutil、bcprov（公钥/私钥、底层算法）

### 在 Linux 上查看当前默认

```bash
# 加密一个文件并看实际使用的算法
gpg -c -vv 某个文件 2>&1 | grep -E 'cipher|s2k|hash|using'
```

输出里会看到类似：`using cipher AES`（即 AES-128）、`s2k-digest-algo SHA1`、`s2k-count` 等。

### 若仍无法互解

- 确认两边密码一致（含空格、大小写）。
- 在 Linux 上显式指定与应用一致再试：
  ```bash
  gpg --s2k-cipher-algo AES256 --s2k-digest-algo SHA1 -c 文件
  ```
- 解密时用 `getDataStream()` 只读 literal 内容，避免把 MDC 当明文读出导致尾部乱码（应用已按此实现）。
